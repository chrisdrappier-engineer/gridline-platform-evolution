# Admin User Stories

## Role

The admin is an internal Gridline user responsible for configuration,
operational oversight, and permission auditing.

## Stories

### Inspect Role Assignments

As an admin, I can inspect user role assignments so permissions can be audited.

Status: Implemented

### Review Permission Matrix

As an admin, I can review the role-to-permission matrix so I can understand what
each role is allowed to do.

Status: Implemented

### Manage Baseline Configuration

As an admin, I can review customers, sites, providers, users, and role
assignments so the operational baseline can be maintained.

Status: Implemented

### Manage Service Quotes And Cost Entries

As an admin, I can create, review, update, or correct service quotes and cost
entries so financial reporting and approval workflows remain accurate.

Status: In Progress

### Audit Request Notes And Files

As an admin, I can review notes and uploaded files across requests so I can
support compliance, troubleshooting, and operational oversight.

Status: Proposed

### Review Permission Impact On Reporting

As an admin, I can verify which roles can see cost, performance, notes, ratings,
and files so reporting data is exposed only to appropriate users.

Status: Proposed
