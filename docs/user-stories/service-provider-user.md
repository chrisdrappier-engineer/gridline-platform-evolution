# Service Provider User Stories

## Role

The service provider user is a provider-side user responsible for tracking
assigned service requests. In the baseline workflow, providers communicate
updates outside the application and Gridline dispatchers record request changes.
Direct provider updates are a future portal capability.

## Stories

### View Assigned Provider Requests

As a service provider user, I can view requests assigned to my provider so I can
see work requiring action.

Status: Implemented

### Record Actions Taken

As a service provider user, I can update a request with actions taken so
Gridline has a record of service work.

Status: Deferred

### Identify Follow-Up Requirements

As a service provider user, I can identify follow-up requirements so unresolved
work does not disappear.

Status: Deferred

### Mark Provider Work Complete

As a service provider user, I can mark provider-side work complete so Gridline
and the facility manager can review the result.

Status: Deferred

### View Provider Performance

As a service provider user, I can view response time, completion time,
resolution percentage, and customer rating trends for my provider organization
so I can understand service quality.

Status: Proposed

### Submit Provider Work Note

As a service provider user, I can add notes about work performed so Gridline and
authorized customer users can see provider-side context without relying on phone
or email.

Status: Proposed

### Upload Provider File

As a service provider user, I can upload invoices, photos, diagnostics, or work
orders to a request so supporting documentation is available for review.

Status: Proposed

### Submit Service Charge

As a service provider user, I can submit charge details for work performed so
Gridline can review service cost before reporting or billing.

Status: Proposed
