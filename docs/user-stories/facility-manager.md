# Facility Manager User Stories

## Role

The facility manager is a customer-side site user responsible for tracking
service request lifecycle, cost, and quality for assigned facilities. In the
baseline workflow, facility managers report issues outside the application and
Gridline dispatchers create or update requests.

## Stories

### View Managed Facility Requests

As a facility manager, I can view service requests for facilities I manage so I
can track open and recently completed work.

Status: Implemented

### Create Request For Managed Facility

As a facility manager, I can create a service request for one of my managed
facilities so I can report maintenance issues without selecting unrelated
customer sites.

Status: Deferred

### Verify Completion

As a facility manager, I can verify completion of a resolved request so Gridline
knows the customer-side issue is actually closed.

Status: Deferred

### View Site Request History

As a facility manager, I can view service request history for a managed facility
so I can understand recurring issues at that site.

Status: Implemented

### View Site Service Costs

As a facility manager, I can view service costs for requests tied to facilities
I manage so I can understand spend at the site level.

Status: Proposed

### View Request Lifecycle Timing

As a facility manager, I can see when a request was reported, assigned,
responded to, completed, and verified so I can understand whether work is moving
at an acceptable pace.

Status: Proposed

### Rate Completed Service

As a facility manager, I can rate completed service and leave a customer-facing
comment so Gridline can track service quality by provider and site.

Status: Proposed

### Upload Site Evidence

As a facility manager, I can upload photos or documents related to a request so
Gridline can review supporting evidence during coordination and reporting.

Status: Proposed
