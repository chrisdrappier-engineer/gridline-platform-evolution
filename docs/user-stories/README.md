# User Stories

This directory captures role-centered user stories for Gridline Platform
Evolution.

These stories bridge the project ADRs and the implementation backlog. They are
not meant to be exhaustive product requirements. They describe the workflow
surface needed to make the mature Rails monolith feel operationally believable
before the project moves into horizontal scaling scenarios.

## Roles

- [Dispatcher](dispatcher.md)
- [Facility Manager](facility-manager.md)
- [Customer Contact](customer-contact.md)
- [Service Provider User](service-provider-user.md)
- [Admin](admin.md)

## Story Status

Use these lightweight status labels when needed:

- Proposed: story is identified but not yet designed in detail
- Ready: story is clear enough to implement
- In Progress: implementation has started
- Done: behavior exists and is verified
- Deferred: story is valid but intentionally outside the current phase
