# Decision Notes

Decision notes summarize substantial planning conversations that shaped project
decisions.

They are not Architecture Decision Records. ADRs remain the source of truth for
final architecture decisions. Decision notes explain how a decision emerged when
the discussion itself is useful context for reviewers.

Use a decision note when:

- an ADR or major architecture direction changes
- AI-assisted reasoning materially shaped the outcome
- the maintainer challenged or redirected important assumptions
- the reasoning path would be useful to understand later

For small commits or routine implementation work, note AI assistance in the
commit message or pull request body instead of adding a decision note.
