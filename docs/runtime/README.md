# Runtime Documentation

Gridline uses Docker Compose for both development and production-like local
execution.

- [Quick Start](quick-start.md) covers the shortest useful local path through
  verification, production-like runtime startup, Scenario 00 workload execution,
  and first-pass result inspection.
- [Development Runtime](development.md) covers the bind-mounted Rails
  development environment, local database setup, CI, and browser workflow
  testing.
- [Production-Like Runtime](production-like.md) covers the workload target,
  production Docker build, resource envelope, production smoke checks, and
  provision/deploy/evidence-reset modes.
